# Подключение Wildberries по API (через GitHub Actions)

Эта связка раз в несколько часов сама забирает заказы и выкупы из Wildberries,
пересчитывает их в формат дашборда и обновляет `data.json` в репозитории.
GitHub Pages сразу отдаёт свежие цифры. Сервер поднимать не нужно, токен в коде
не светится — он лежит в защищённых секретах репозитория.

## Файлы

```
wb_sync.py                       скрипт: WB Statistics API → data.json
.github/workflows/wb-sync.yml    запуск по расписанию + ручной, коммит data.json
```

Положите оба в репозиторий: `wb_sync.py` — в корень (рядом с `data.json`),
`wb-sync.yml` — по пути `.github/workflows/wb-sync.yml`.

## Шаг 1. Получить токен статистики Wildberries

1. Войдите в кабинет продавца: `https://seller.wildberries.ru`.
2. Профиль (ФИО справа вверху) → **Настройки → Доступ к новому API**.
3. Создайте токен и выберите тип/категорию **«Статистика»**. Имя — любое
   (например, `bioboro-dashboard`).
4. Скопируйте токен сразу — он показывается один раз.

Токен типа «Статистика» отдаёт только данные о продажах и заказах и не даёт
доступа к управлению кабинетом.

## Шаг 2. Положить токен в секреты репозитория

В репозитории: **Settings → Secrets and variables → Actions → New repository
secret**.
- **Name:** `WB_STATS_TOKEN`
- **Secret:** вставьте скопированный токен → **Add secret**.

Имя секрета должно быть ровно `WB_STATS_TOKEN` — на него ссылается workflow.

## Шаг 3. Включить Actions и запустить первый раз

1. Вкладка **Actions** в репозитории — если просит подтвердить включение
   workflow’ов, согласитесь.
2. Слева выберите **WB sync** → справа кнопка **Run workflow** → **Run workflow**.
3. Откройте запущенный прогон: в логе шага *Pull Wildberries data* увидите,
   сколько заказов и продаж пришло и какая получилась выручка. Шаг
   *Commit data.json* запушит обновлённый файл.

После этого дашборд (локально, или на GitHub Pages) покажет реальные цифры WB.

## Расписание

По умолчанию — каждые 6 часов. Поменять можно в `wb-sync.yml`, строка `cron`
(время в UTC). Примеры:
- `"0 */3 * * *"` — каждые 3 часа;
- `"0 5 * * *"` — раз в сутки в 05:00 UTC (08:00 МСК).

GitHub иногда запускает запланированные задачи с задержкой в несколько минут —
это нормально. Ручной запуск (Run workflow) срабатывает сразу.

## Что попадает в data.json

Период синхронизации — **текущий и прошлый месяц** (чтобы работали сравнение
МоМ и прогноз). Строки канала `Wildberries` за это окно перезаписываются
свежими, остальные каналы и блок `plans` не трогаются.

Из API статистики заполняются:
- `revenue_orders` — выручка по заказам (сумма `priceWithDisc`, без отменённых);
- `orders_count` — число заказов (без отменённых);
- `revenue_buyouts` — выручка по выкупам (продажи минус возвраты);
- `buyouts_count` — число выкупов;
- `commissions` — комиссия WB (≈ `priceWithDisc − forPay` по выкупам);
- `net_revenue` и `margin` — считаются автоматически.

Поля `ad_spend`, `logistics`, `penalties`, `other_costs` ставятся в `0` —
их в статистическом API нет. Когда захотите видеть их по факту, есть два пути:
1. дозаполнять вручную в `data.json`;
2. расширить `wb_sync.py` запросом к финансовому отчёту
   `/api/v5/supplier/reportDetailByPeriod` (нужен токен категории «Финансы»),
   откуда берутся комиссии, логистика, хранение и штрафы построчно. Скажите —
   добавлю этот блок в скрипт.

## План на месяц

Скрипт **не трогает планы**. Цели по выручке/заказам ставьте как раньше —
в ячейках дашборда или в разделе `plans` файла `data.json`.

## Прочие каналы

Автоматически тянется только Wildberries. Ozon, «Сайт», VK, YouTube, Clinics
остаются как есть (тестовые или внесённые вами). Ozon можно подключить таким же
способом отдельным шагом — по запросу подготовлю.

## Если что-то не так

- **HTTP 401** в логе — токен неверный/просрочен или не той категории. Пересоздайте
  токен «Статистика» и обновите секрет `WB_STATS_TOKEN`.
- **HTTP 429** — сработал лимит запросов WB (примерно 1 запрос в минуту на
  эндпоинт). Скрипт делает паузы; при повторе обычно проходит. Можно увеличить
  паузу переменной `WB_REQUEST_INTERVAL` в шаге запуска.
- **Шаг Commit ничего не пушит** — значит данные не изменились с прошлого раза,
  это нормально.
