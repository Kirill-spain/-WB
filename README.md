# BIOBORO — трекер выполнения плана

Лёгкий управленческий дашборд: открываешь страницу и сразу видишь, идёшь ли в план,
насколько отстаёшь, сколько нужно добрать, какой прогноз на месяц, какой канал
проседает и что происходит по сравнению с прошлым месяцем.

Чистый стек: **HTML + CSS + ванильный JavaScript + Chart.js**. Без сборки и бэкенда.

---

```
bioboro/
```

## Быстрый старт

### Вариант A. Автономный файл (без сервера)
Откройте **`bioboro-dashboard-standalone.html`** двойным кликом. В нём встроены данные и
стили — нужен только интернет для подгрузки Chart.js с CDN. Удобно «просто посмотреть».

### Вариант B. Многофайловый проект (рекомендуется для работы)
Браузеры блокируют чтение локальных файлов через `file://`, поэтому `index.html` нужно
открывать через локальный веб-сервер:

```bash
cd bioboro
python3 -m http.server 8000
# затем откройте http://localhost:8000
```

Любой статический сервер подойдёт (`npx serve`, расширение Live Server в VS Code и т. п.).

## Расчёты (формулы)

| Показатель | Формула |
|---|---|
| Дневной план | `monthly_plan / days_in_month` |
| Накопительный план к дате | `daily_plan * current_day` |
| Выполнение плана, % | `actual_cumulative / cumulative_plan_today * 100` |
| Выполнение месяца, % | `actual_cumulative / monthly_plan * 100` |
| Средний темп/день | `actual_cumulative / current_day` |
| Прогноз на месяц | `daily_average_actual * days_in_month` |
| Разрыв прогноза | `month_forecast - monthly_plan` |
| Buyout rate, % | `buyouts_count / orders_count * 100` |
| Средний чек | `revenue_orders / orders_count` |
| ДРР, % | `ad_spend / revenue_orders * 100` |
| Чистая выручка | `revenue_buyouts - ad_spend - logistics - commissions - penalties - other_costs` |

«Текущий день» для текущего месяца — сегодняшнее число; для прошедшего месяца — это
число дней в месяце (месяц закрыт); для будущего — 0. Сравнение «месяц к месяцу» берёт
одинаковый диапазон дней (например, 1–10 июня против 1–10 мая).

## Схема данных (`data.json`)

```json
{
  "meta": {
    "business": "BIOBORO",
    "currency": "RUB",
    "default_month": "2026-06",
    "default_metric": "revenue_buyouts",
    "channels": ["Wildberries", "Ozon", "Site", "Clinics", "VK", "YouTube"]
  },
  "plans": {
    "2026-06": {
      "revenue_buyouts": 10000000,
      "revenue_orders": 12500000,
      "orders_count": 7353,
      "buyouts_count": 5882,
      "net_revenue": 4200000
    },
    "2026-05": { "...": "..." }
  },
  "data": [
    {
      "date": "2026-06-01",
      "channel": "Wildberries",
      "revenue_orders": 208340,
      "revenue_buyouts": 160810,
      "orders_count": 133,
      "buyouts_count": 104,
      "ad_spend": 34160,
      "logistics": 11630,
      "commissions": 29050,
      "penalties": 1930,
      "other_costs": 1610,
      "net_revenue": 82430,
      "margin": 53.5
    }
  ]
}
```

* `plans` — месячные планы по каждой метрике, ключ месяца в формате `YYYY-MM`.
* `data` — одна запись на день на канал. Дашборд сам агрегирует по дням и каналам.
* Поля строки: `date`, `channel`, `revenue_orders`, `revenue_buyouts`, `orders_count`,
  `buyouts_count`, `ad_spend`, `logistics`, `commissions`, `penalties`, `other_costs`, `net_revenue`, `margin`.

Чтобы добавить месяц — допишите план в `plans` и строки в `data`.
Перегенерировать пример целиком:

```bash
python3 generate_data.py     # перезапишет data.json (детерминированно, seed фиксирован)
```

## Выбор даты и ввод плана

**Календарь «Сегодня (точка отсчёта)».** В шапке — поле с всплывающим календарём.
Им задаётся, какой день считать текущим: меняются «план к дате», отставание, прогноз,
сравнение с прошлым месяцем и подсветка в таблице. По умолчанию берётся `meta.today`
или `meta.generated_at` (а если их нет — системная дата), но не выходя за пределы дат,
на которые есть данные. Выбор даты в другом месяце автоматически переключает и фильтр
месяца.

**Редактор плана.** Карточка «План на месяц» под баннером — ячейки для ввода плана по
каждой метрике (чистая выручка и прочие — в деньгах, заказы/выкупы — в штуках). Правки
применяются сразу и **сохраняются в браузере** (localStorage), поэтому в скачанном файле
и на хостинге сохраняются между перезагрузками. Кнопка «Сбросить к data.json» убирает
ручные правки. Чтобы зафиксировать план навсегда и на всех устройствах — впишите его в
`plans` в `data.json` (или в `DEFAULT_CONFIG.plans` для Google Sheets).

> В предпросмотре внутри чата localStorage может быть недоступен — тогда правки плана
> действуют до перезагрузки страницы. В обычном браузере всё сохраняется.

## Источник данных — переключение

Источник задаётся в начале `app.js` в объекте `SOURCE`:

```js
const SOURCE = {
  type: 'json',                 // 'json' | 'github' | 'gsheets'
  jsonUrl: 'data.json',
  githubRawUrl: 'https://raw.githubusercontent.com/USER/REPO/main/data.json',
  gsheetsCsvUrl: 'https://docs.google.com/spreadsheets/d/SHEET_ID/export?format=csv&gid=0',
};
```

Приоритет загрузки: если на странице задан `window.BIOBORO_DATA` (как в автономном
файле), берутся встроенные данные. Иначе используется `SOURCE.type`.

### 1) Локальный `data.json` (по умолчанию)
`type: 'json'` — файл рядом с `index.html`. Ничего настраивать не нужно.

### 2) GitHub (raw JSON)
1. Залейте `data.json` в репозиторий.
2. Откройте файл на GitHub → кнопка **Raw** → скопируйте ссылку вида
   `https://raw.githubusercontent.com/USER/REPO/BRANCH/data.json`.
3. В `app.js`: `type: 'github'` и вставьте ссылку в `githubRawUrl`.

Так дашборд всегда тянет свежие данные из репозитория — правите JSON, обновляете страницу.

### 3) Google Sheets (опубликованный CSV)
В этом режиме лист содержит **только дневные строки**; `meta` и `plans` берутся из
константы `DEFAULT_CONFIG` в `app.js` (отредактируйте планы там).

1. Сделайте таблицу, где первая строка — заголовки ровно с такими именами столбцов:
   `date, channel, revenue_orders, revenue_buyouts, orders_count, buyouts_count,
   ad_spend, logistics, commissions, penalties, other_costs, net_revenue, margin`
   (`date` в формате `YYYY-MM-DD`).
2. **Файл → Поделиться → Опубликовать в интернете → формат CSV.** Либо используйте
   ссылку экспорта:
   `https://docs.google.com/spreadsheets/d/SHEET_ID/export?format=csv&gid=GID`
   (`SHEET_ID` — из адреса таблицы, `GID` — id листа из адресной строки).
3. В `app.js`: `type: 'gsheets'` и вставьте ссылку в `gsheetsCsvUrl`.

Встроенный CSV-парсер понимает кавычки и числа с пробелами/запятой-разделителем.

## Что показывает дашборд

* **Баннер-вердикт** — «иду в план / отстаю», выполнение %, сколько добрать сейчас,
  нужный темп до конца месяца.
* **KPI** — план/факт месяца и на сегодня, отклонение ₽ и %, прогноз, заказы, выкупы,
  средний чек, ДРР, чистая выручка.
* **Прогресс-бар месяца** с отметкой «план к сегодняшнему дню».
* **График** накопительный план vs факт vs прогноз.
* **Каналы** — текущий период против прошлого месяца, проседающие подсвечены.
* **Сравнение МоМ** — одинаковый диапазон дней этого и прошлого месяца.
* **Таблица по дням** с подсветкой опережения/отставания.

Фильтры: месяц, канал продаж, метрика (выручка по выкупам/заказам, чистая выручка,
заказы, выкупы). Вёрстка адаптивная — работает с телефона.
